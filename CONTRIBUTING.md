# Contributing

1. Create a feature branch from `main`.
2. Run `make validate` and `make policy` before opening a merge request.
3. Do not commit secrets, kubeconfigs, private keys, production passwords, customer data, internal inventories, or confidential disclosure material.
4. Use `example.com`, `.example` names, and RFC 5737 documentation IP ranges for examples instead of real environments.
5. Keep deployment changes configuration-driven where possible.
6. Update `docs/release-guide.md` for release-affecting changes.
